# IRENA Renewable Energy Policy Tracker

## What this project is
Interactive policy tracker for IRENA ÔÇö a world map + filterable table showing renewable energy policy and market responses to the 2026 Iran war energy crisis. Built as a static HTML demo hosted on GitHub Pages, with future integration into IRENA's Sitecore CMS (likely via iframe).

**Live site:** https://nadeemgoussous.github.io/irena-policy-tracker/
**GitHub repo:** https://github.com/nadeemgoussous/irena-policy-tracker

## Key files

| File | Role |
|------|------|
| `index.html` | Standalone tracker (own IRENA-branded header/footer) ÔÇö **this is the GitHub Pages entry point** |
| `irena_policy_tracker.html` | Tracker embedded inside the real IRENA site template ÔÇö generated by `build_tracker.py`, do not edit directly |
| `build_tracker.py` | Script that generates `irena_policy_tracker.html` from `webpage_design.html` + tracker logic ÔÇö kept for future Sitecore sync, but **primary development happens in `index.html`** |
| `data.json` | Policy database ÔÇö **edit this to add/update entries** |
| `policies.md` | Source document for current dataset (2026 Iran crisis responses) |
| `world-110m.json` | TopoJSON world map geometry (Natural Earth 110m, do not edit) |
| `webpage_design.html` | Saved IRENA site page used as design template ÔÇö **not in git, do not commit** |

## Running locally

```bash
cd policy_tracker
python -m http.server 8080
# open http://localhost:8080/index.html
```

Requires a server (not `file://`) because `data.json` and `world-110m.json` are fetched via `fetch()`.

## Deploying changes

```bash
# After editing index.html or data.json:
git add index.html data.json
git commit -m "your message"
git push
# GitHub Pages auto-deploys from main branch root (~1 min)
```

If you need to sync changes to the Sitecore-embedded version:
```bash
python build_tracker.py
git add irena_policy_tracker.html build_tracker.py
```
Note: `irena_policy_tracker.html` is currently behind `index.html` ÔÇö it does not have the org panel, context toggle, or full-text search.

## Current data schema

Each entry in `data.json`:
```json
{
  "id": 1,
  "country": "Spain",
  "iso3": "ESP",
  "region": "Europe",
  "policy_name": "RE Grid as Energy Security Shield",
  "sector": "Cross-cutting",
  "policy_type": "Existing Capacity",
  "date": "Ongoing March 2026",
  "quantitative_details": "60%+ electricity from renewables; wholesale prices ÔéČ37ÔÇô57/MWh",
  "context": "Decades of RE investment now paying off as energy security shield"
}
```

Valid `region` values: `Africa` / `Americas` / `Asia-Pacific` / `Europe` / `International` / `Middle East`

Valid `sector` values: `Biofuels` / `Conservation` / `Cross-cutting` / `Energy Efficiency` / `Grid / Storage` / `Hydro` / `Nuclear` / `Solar` / `Transport / EVs` / `Wind`

Valid `policy_type` values: `Analysis` / `Emergency Measure` / `Existing Capacity` / `Financing / Strategy` / `Incentive` / `Infrastructure` / `Mandate / Regulation` / `Market Response` / `Target / Strategy`

For multilateral organizations (IEA, IRENA, ASEAN, G7 etc.) use `iso3: "INT"` and `region: "International"`.

## UI features (current)

**Table columns:** Country / Org ┬Ě Region ┬Ě Policy / Measure (with expandable context) ┬Ě Sector ┬Ě Type (colour badge) ┬Ě Date ┬Ě Key Details

**Filter dropdowns:** Region ┬Ě Sector ┬Ě Policy Type ┬Ě Search (full-text: searches country, policy name, sector, type, key details, context, date)

**Map:** D3 choropleth ÔÇö click a country to filter the table; choropleth shading reflects current filtered counts

**Org panel:** Clickable chip strip below the map for multilateral/regional orgs (`iso3: "INT"` or `"EUU"`) that have no map geometry ÔÇö IEA, IRENA, EU, ASEAN, G7, UNFCCC, Ember, IEEFA

**Context toggle:** Each table row has a `ÔľŞ context` button that expands an inline quote/context block

## Layout constraints

The tracker in `irena_policy_tracker.html` sits inside IRENA's `l-grid__m--10` content column (10/12 of 1316px Ôëł 1097px wide) alongside the Newsletter sidebar. Do not break it out to full width.
